# ПМИ
## 1.Объект испытаний
- программа, для загрузки fbt файлов по TCP в FORTE
## 2. Цепь испытаний
- 2.1 Персональный компьютер под управлением ОС Linux или Windows.
- 2.2 Среда исполнения FORTE, запущенная в режиме TCP‑сервера
(порт 61499).
- 2.3 Испытуемая программа (TCP‑клиент загрузки FBT/FBOOT).
- 2.4 Средства мониторинга и отладки:
- 2.5 wireshark для анализа сетевого протокола;
- 2.6 zzuf для фаззинг‑тестирования;
- 2.7 gdb для анализа аварийных завершений (core dump).
## 3 Требования к программе
- 3.1 Корректно устанавливать TCP‑соединение с FORTE по заданному адресу и порту.
- 3.2 Обеспечивать передачу XML‑команд.

## 4 Требования к программной документации
- 4.1 Техническое задание
- 4.2 Архитектура программы
- 4.3 Программа и методика испытаний

## 5 Состав и порядок испытаний

* **5.1 Функциональные испытания**
- Запуск FORTE в режиме ожидания TCP‑подключений.
- Запуск испытуемой программы.
- Проверка успешного подключения к FORTE.
- Передача корректной XML‑команды CREATE.
- Контроль реакции FORTE.

* **5.2 Фаззинг‑тестирование**
- Запуск программы под управлением zzuf.
- Искажение входных параметров и передаваемых данных.
- Фиксация сбоев и аварийных завершений.
